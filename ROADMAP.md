### 陆家花园项目发展路线图 (Roadmap) - 修订版

> **项目背景**：陆家花园诗歌元宇宙项目由诗人西尔（《毛小豆故事演绎》作者，也是诗人吴任几的朋友）发起，旨在通过数字技术重新诠释和体验诗歌。项目包含两个核心子项目：周与春秋练习项目（已发布v2.0.0）和毛小豆宇宙项目（开发中v3.0）。

> **开发者背景**：非专业开发者，从2025年7月26日开始学习编程，主要依赖Cursor AI辅助开发。项目采用严格的Git开发规范和AI协作指南。

#### **总览**

本项目的发展将分为三个核心阶段，**基于现有架构进行渐进式演进**，而非重新构建。目标是整合两个子项目的数据和服务，最终实现一个统一的诗歌元宇宙体验平台。

*   **第一阶段：数据整合与数据库化** - 让两个子项目"融合"。基于现有的Express.js后端，建立统一的数据库系统，将毛小豆宇宙的数据有机融入陆家花园主宇宙，实现数据的深度整合。
*   **第二阶段：前端体验升级** - 让界面"活"起来。采用渐进式升级策略，在保持现有功能稳定的基础上，逐步引入现代前端技术。
*   **第三阶段：功能深化与体验优化** - 让诗歌"立体"起来。在稳固的技术基础上，探索更丰富的诗歌互动体验。

---

## **第一阶段：数据整合与数据库化**

**目标**：基于现有的 `poeject_zhou_spring_autumn/application/server.js`，建立统一的数据库系统，将毛小豆宇宙的数据有机融入陆家花园主宇宙，实现两个子项目数据的深度整合。

### **子阶段划分**

#### **子阶段 A：数据库设计与数据建模** 
- **优先级**: P0
- **预计时长**: 5-7 天（考虑学习时间）
- **关键风险**: 数据库设计概念理解、数据关系建模
- **回滚方案**: 保持现有JSON文件作为数据源
- **学习资源**: SQLite基础、数据库设计原则、数据建模

**任务列表**:
- [ ] 分析两个子项目的数据结构和关系
- [ ] 设计统一的数据库表结构，体现"陆家花园主宇宙"概念
- [ ] 建立数据关联关系（诗歌-角色-场景-主题的关联）
- [ ] 安装和配置SQLite + Prisma环境

**退出条件**: 
- 数据库表结构设计完成，体现主宇宙概念
- 数据关系模型清晰明确
- 开发环境配置完成

#### **子阶段 B：数据迁移与整合** 
- **优先级**: P0  
- **预计时长**: 4-6 天
- **关键风险**: 数据迁移完整性、数据一致性保证
- **回滚方案**: 切换回JSON文件数据源
- **学习资源**: 数据迁移策略、SQL操作、数据验证

**任务列表**:
- [ ] 编写数据迁移脚本，将JSON数据导入数据库
- [ ] 建立毛小豆宇宙与周春秋项目的数据关联
- [ ] 实现数据完整性验证和一致性检查
- [ ] 创建数据统计和分析功能

**退出条件**:
- 所有JSON数据成功迁移到数据库
- 两个子项目数据关联建立完成
- 数据完整性验证通过

#### **子阶段 C：统一API网关与数据服务** 
- **优先级**: P1
- **预计时长**: 5-7 天  
- **关键风险**: API设计、数据库查询性能、数据访问权限
- **回滚方案**: 切换回子阶段B的稳定版本
- **学习资源**: RESTful API设计、数据库查询优化、缓存策略

**任务列表**:
- [ ] 重构现有API，使用数据库替代JSON文件
- [ ] 编写API自动化测试（使用Jest/Supertest），确保重构前后接口行为一致
- [ ] 创建统一的API网关，支持跨项目数据访问
- [ ] 实现数据查询和关联查询功能
- [ ] 添加数据缓存和性能优化机制

**退出条件**:
- 统一API网关正常工作
- 跨项目数据查询功能完善
- 查询性能满足要求

### **第一阶段整体退出条件**
- 毛小豆宇宙数据成功融入陆家花园主宇宙

- 数据一致性和完整性得到保证

## **学习支持与风险控制**

### **学习资源**
- **编程基础**: MDN Web Docs、JavaScript.info
- **框架学习**: Vue.js官方文档、Express.js指南
- **设计资源**: Bootstrap文档、CSS Grid/Flexbox教程
- **问题解决**: Stack Overflow、GitHub Issues

### **常见问题与解决方案**
- **API设计困惑**: 参考RESTful API设计原则
- **异步编程困难**: 从Promise开始，逐步学习async/await
- **样式调整复杂**: 使用浏览器开发者工具调试
- **部署问题**: 先确保本地运行正常，再考虑部署

### **风险控制**
- **版本控制**: 为主要阶段创建特性分支（如 `feature/phase-1`），将变更与主干隔离，确保主干始终稳定。
- **技术风险**: 每个阶段都有明确的回滚方案
- **学习风险**: 预留充足的学习时间，不急于求成
- **项目风险**: 保持现有功能的稳定性，渐进式改进

### **求助机制**
- **AI助手**: 充分利用Cursor AI的编程辅助功能
- **社区支持**: 在遇到问题时寻求技术社区帮助
- **文档记录**: 及时记录学习过程和解决方案
